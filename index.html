How "not" to store passwords

Store password as a last resort

You are not safe

The naive way #1 - storing plain text passwords in the database

Register

user = User()
user.email = 'someguy@example.com'
user.password = 'qwer1234'
user.save()

Login

if user.password == passwordInput
    return True
else
    return False

Email Password
someguy@example.com qwer1234
... ...

The naive way #2 - Encrypting

Register

key = 'secret'
user = User()
user.email = 'someguy@example.com'
user.password = encrypt(key,'qwer1234')
user.save()

Login

if decrypt(key,user.password) == passwordInput
    return True
else
    return False

Email Password
someguy@example.com O2jxMx3eyqE=
... ...

Problems

  1. Keys can be stolen
  2. identical password will have the same encrypted form

Example

Email Password Hint
michael@example.com O2jxMx3eyqE= Song
sam@example.com O2jxMx3eyqE= Michael Jackson
bob@example.com O2jxMx3eyqE= Horror
steve@example.com O2jxMx3eyqE= Scary michael

The naive way #3 - Hashing

Register

user = User()
user.email = 'someguy@example.com'
user.password = hash('qwer1234')
user.save()

Login

if user.password == hash(passwordInput)
    return True
else
    return False

Email Password
someguy@example.com 9a3bdc603c940a18467d167af15d4c8c
... ...

Problems

  1. can be cracked using brute force
  2. can be cracked using hash tables/rainbow tables
  3. identical password will have the same hash

Safer way - Hash and Salt

Register

user = User()
user.salt = random_string()
user.email = 'someguy@example.com'
user.password = hash('qwer1234'+user.salt)
user.save()

Login

if user.password == hash(passwordInput+user.salt)
    return True
else
    return False

Email Password
someguy@example.com ZKeymuvG9HtbaTdlZ6KDA.WT2LjfrXaazaAZa7QX7tjcnY/wxvHB6
someguy.with.same.password@example.com I.2rNtKY7K9PXBldLSl94uWAUMwUIf1T8i.BtiNWnkNvZEhC9BSsq
... ...

Pros

  1. you cannot decrypt it
  2. same passwords have different hashes
    • Cannot be guessed
    • Cannot reverse using hash/rainbow tables
